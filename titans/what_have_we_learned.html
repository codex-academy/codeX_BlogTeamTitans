GIT

GitHub's a great tool but it's definitely a little confusing the first time around.

We have learned that git works by reading a local code repository on your computer and mirroring that code elsewhere. Then we commit our entire local repository to GitHub. As we continue to work on our code, we'll simply commit changes made. GitHub will then keep track of the changes we made, creating different versions of files so we can revert back to old ones if we want. When multiple developers commit code with git, GitHub becomes a central repository where all the code that everyone's working on can stay in sync. We'll commit our changes, and other developers will pull them. We'll do the same with their code.

Git makes this all happen.
